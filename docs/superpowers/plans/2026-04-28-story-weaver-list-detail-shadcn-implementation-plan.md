# Story Weaver List-Detail shadcn Unification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the renderer into a `shadcn/ui`-aligned desktop shell with left navigation, a list-detail `作品` workspace, and standalone `新建作品` and `设置` views without changing the underlying IPC-driven writing flows.

**Architecture:** Keep the existing Electron, IPC, and storage logic intact and concentrate changes inside the renderer. First, make the codebase compatible with official `shadcn` component generation and add the missing primitives. Then introduce a top-level shell with sidebar navigation, move the book workspace to a master-detail layout, and finally reshape `NewBook` and `Settings` into standalone views while updating behavior-based renderer tests.

**Tech Stack:** Electron, React 19, TypeScript, Vite 6, Tailwind CSS 4, `shadcn/ui`, Radix UI, lucide-react, Vitest, Testing Library

---

## File Structure

Create and keep these responsibilities:

- `package.json`: add the missing runtime packages needed by official `shadcn` components.
- `tsconfig.json`: add the `@/*` path mapping that points at `renderer/*`.

- `components.json`: keep `style: default` and `baseColor: slate`, but ensure it matches the actual alias layout used by generated components.

- `renderer/components/BookCard.tsx`: convert from card-grid item to selectable list row.

- `renderer/pages/NewBook.tsx`: standalone creation view with `shadcn` layout rhythm.
- `renderer/pages/Settings.tsx`: standalone settings view with clearer list/edit sections.

Delete or stop importing obsolete files once the new structure is in place:

- `renderer/pages/Library.tsx`: `作品` list-detail workspace that replaces the old stacked shelf view.
